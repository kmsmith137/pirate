AssembledFrame¶

class pirate_frb.core.AssembledFrame¶

Data frame containing beamformed data for one (time_chunk, beam_id) pair.

The data is stored in its raw binary format (int4, special value -8 indicates masked data, float16 offsets/scales at lower resolution). Most python callers will want to convert to float32 by calling dequantize().

property beam_id¶

Beam id of this frame.

property data¶

Raw int4 data (pre-scaled) as uint8 array with shape (nfreq, ntime/2).

Most python callers will want to call dequantize(), rather than working directly with the raw data.

dequantize(self: pirate_frb.pirate_pybind11.AssembledFrame) → Array<float>¶

Convert raw data/scales/offsets to a float32 array of shape (nfreq, ntime).

NOTE: we currently convert masked samples (represented by raw int4 value -8) to zeroes. We don’t currently define a separate method to return the boolean mask, but if you need this let me know.

property fpga_seq_end¶

FPGA sequence number one-past-the-end of this frame’s time chunk (= start of the next chunk).

property fpga_seq_start¶

FPGA sequence number at the start of this frame’s time chunk (= time_chunk_index * ntime * metadata.seq_per_frb_time_sample).

static from_asdf(filename: str) → pirate_frb.pirate_pybind11.AssembledFrame¶

Read an AssembledFrame back from an ASDF file.

Note: XEngineMetadata is “projected” through ASDF – after reading, the metadata’s beam_ids / beam_positions_{x,y} are length-1 (just this frame’s beam) and freq_channels is empty.

static make_uninitialized(xmd: pirate::XEngineMetadata, ntime: SupportsInt | SupportsIndex, beam_id: SupportsInt | SupportsIndex, time_chunk_index: SupportsInt | SupportsIndex) → pirate_frb.pirate_pybind11.AssembledFrame¶

Create an AssembledFrame backed by the given XEngineMetadata, with freshly-allocated data / scales_offsets arrays whose CONTENTS ARE UNSPECIFIED – the caller must fill them (e.g. via randomize()).

Throws if xmd is None or if beam_id is not in xmd.beam_ids. nfreq is taken from xmd.get_total_nfreq().

xmd.freq_channels is ignored. Callers should typically pass a frequency-scrubbed xmd so the returned frame matches the ‘always frequency-scrubbed’ invariant on AssembledFrame.metadata.

property metadata¶

Shared XEngineMetadata for this frame. Read-only by convention – do not mutate the returned object, since it is shared with all sibling frames from the same allocator.

Always frequency-scrubbed: metadata.freq_channels is empty.

property nfreq¶

Number of frequency channels.

property ntime¶

Number of time samples in this frame’s time chunk.

randomize(self: pirate_frb.pirate_pybind11.AssembledFrame, normalize: bool, gaussian: bool, sp: pirate_frb.pirate_pybind11.SinglePulse = None, dt_sp: SupportsInt | SupportsIndex = 0) → None¶

Fill the frame with random int4 data (intended for testing), optionally injecting a simulated FRB pulse.

If gaussian=False, data is uniform over [-8, +7]. If gaussian=True, data is simulated Gaussian noise quantized to int4 and clamped to [-7, +7] (the -8 sentinel is never produced).

If normalize=False, scales are uniform in [0,1] and offsets in [-1,1]. If normalize=True, then offsets are zero, and scales are chosen to match metadata.noise_variance.

sp (default None): a simulated FRB to add on top of the noise.

property scales_offsets¶

Raw scales/offsets as float16 array with shape (nfreq, ntime/256, 2), where the last axis is {scale, offset}.

Most python callers will want to call dequantize(), rather than working directly with the scales/offsets.

property time_chunk_index¶

Index of this frame’s time chunk (one “chunk” is ntime samples).

write_asdf(self: pirate_frb.pirate_pybind11.AssembledFrame, filename: str, sync: bool = True, verbose: bool = False) → None¶

Write this AssembledFrame to an ASDF file.

If sync=True (default), fsync() the file after writing to avoid runaway page cache usage.

If verbose=True, emit comments throughout the YAML header. The comments are detailed enough that the header serves as self- contained documentation of the file format. Intended for use with ‘pirate_frb show_file_format’.
