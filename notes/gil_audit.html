GIL audit for pybind11 wrappers¶

Background¶

pybind11’s py::call_guard<py::gil_scoped_release>() releases the GIL during the C++ function body, allowing other Python threads to run concurrently. The sequence is:

  1. Arguments converted (GIL held – type_caster runs here)

  2. GIL released

  3. C++ function executes

  4. GIL re-acquired

  5. Return value converted (GIL held – type_caster runs here)

ksgpu::Array type_caster: Both load() (Python->C++) and cast() (C++->Python) call Python/numpy C API (PyObject_GetAttrString, PyArray_New, PyImport_ImportModule, Py_INCREF, etc.) and require the GIL. However, call_guard<gil_scoped_release> is safe to use with Array arguments/returns – the GIL is only released during step 3.

Lambdas that call pybind11 API: A few wrappers use lambdas that call py::make_tuple() or py::object methods. These need the GIL during the lambda body, so call_guard cannot be used directly. To drop the GIL in these cases, manually scope py::gil_scoped_release around just the C++ call.
