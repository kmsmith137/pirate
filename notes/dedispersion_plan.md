# Dedispersion plan (search -> grouper)

When the FRB search process connects to its co-located grouper, the first
message of the gRPC `Session` stream is a handshake that carries three YAML
documents (see [`grpc/frb_grouper.proto`](../grpc/frb_grouper.proto) and the
`FrbServer` handshake code):

- `xengine_metadata_yaml` -- the X-engine metadata
  (see [`configs/xengine/xengine_metadata_v2.yml`](../configs/xengine/xengine_metadata_v2.yml));
- `dedispersion_config_yaml` -- the (postfilled) `DedispersionConfig`; and
- `dedispersion_plan_yaml` -- the derived `DedispersionPlan`.

The grouper uses the latter two to interpret the shared GPU output ring buffer
(frequency-zone layout, tree structure, per-tree output shapes) without having
to re-derive any of it.

Note that `dedispersion_config_yaml` is the *postfilled* config: it is the
search node's config file with four metadata-dependent members (`zone_nfreq`,
`zone_freq_edges`, `time_sample_ms`, `beams_per_gpu`) overwritten from the live
X-engine metadata. So it can differ from the on-disk config file if those
values were left unset or stale.

## Example file

[`configs/example_dedispersion_plan.yml`](../configs/example_dedispersion_plan.yml)
is a checked-in example of `dedispersion_plan_yaml` (the third document above).
It is exactly the bare output of `show_dedisperser` (see below), so it is a
single valid YAML document that matches what goes on the wire. It is
auto-generated by the Makefile and serves purely as documentation -- nothing
reads it at runtime. (The config wire field is just the input
[`configs/dedispersion/chord_sb2_et.yml`](../configs/dedispersion/chord_sb2_et.yml)
after postfilling, and the metadata field is documented by
[`configs/xengine/xengine_metadata_v2.yml`](../configs/xengine/xengine_metadata_v2.yml).)

## Regenerating from the command line

The same text is produced by the `show_dedisperser` subcommand:

```
python -m pirate_frb show_dedisperser configs/dedispersion/chord_sb2_et.yml
```

By default this prints only the `DedispersionPlan`, with no separator, matching
the `dedispersion_plan_yaml` wire field; the output is deterministic, which is
why the checked-in snapshot stays byte-stable across rebuilds. Pass `-c` to also
print the `DedispersionConfig` (the `dedispersion_config_yaml` wire field), with
a separator before the plan; `-t` to also print how long plan construction took
(a non-deterministic line); `-v` for inline comments explaining each field; or
see `python -m pirate_frb show_dedisperser -h` for the full set of options.
